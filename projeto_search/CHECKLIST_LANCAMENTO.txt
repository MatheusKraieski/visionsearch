PASSO 1 — Ativar pgvector no Supabase  [CONCLUIDO]
------------------------------------------------------

  Sem isso a busca por similaridade nao funciona e os embeddings
  nao sao salvos nos produtos.

  Como fazer:
    1. Acessar supabase.com > seu projeto
    2. Ir em SQL Editor (menu lateral)
    3. Colar e executar o seguinte SQL:

  -- Cole exatamente isso no SQL Editor:

  CREATE EXTENSION IF NOT EXISTS vector;

  ALTER TABLE products
    ADD COLUMN IF NOT EXISTS embedding vector(384);

  CREATE OR REPLACE FUNCTION match_products(
    query_embedding vector(384),
    match_threshold float DEFAULT 0.3,
    match_count int DEFAULT 10
  )
  RETURNS TABLE (
    id uuid, name text, code text, price numeric,
    image_url text, supplier_id uuid, category text,
    tags text[], similarity float
  )
  LANGUAGE sql STABLE AS $$
    SELECT id, name, code, price, image_url, supplier_id, category, tags,
      1 - (embedding <=> query_embedding) AS similarity
    FROM products
    WHERE 1 - (embedding <=> query_embedding) > match_threshold
    ORDER BY embedding <=> query_embedding
    LIMIT match_count;
  $$;

  -- Fim do SQL

  Como verificar se funcionou:
    Executar no SQL Editor: SELECT * FROM products LIMIT 1;
    Se aparecer a coluna "embedding" na listagem, deu certo.

PASSO 5 — Fazer o deploy na Vercel  [~20 minutos]
--------------------------------------------------

  O projeto ainda so roda localmente. Para acessar de qualquer
  lugar e compartilhar com clientes, precisa estar na Vercel.

  Pre-requisito: codigo precisa estar em um repositorio no GitHub.

  Se o codigo ainda nao esta no GitHub:
    1. Criar repositorio em github.com (pode ser privado)
    2. Na pasta do projeto, rodar:
         git init
         git add .
         git commit -m "primeiro commit"
         git remote add origin https://github.com/seu-usuario/seu-repo.git
         git push -u origin main

  Deploy na Vercel:
    1. Acessar vercel.com e criar conta (usar login do GitHub)
    2. Clicar em "Add New Project"
    3. Importar o repositorio do GitHub
    4. Framework: Vite (detectado automatico)
    5. Em "Environment Variables", adicionar todas as variaveis:

         VITE_SUPABASE_URL       = (copiar do .env.local)
         VITE_SUPABASE_ANON_KEY  = (copiar do .env.local)
         GROQ_API_KEY            = (copiar do .env.local)
         HF_TOKEN                = (copiar do .env.local)

    6. Clicar em Deploy
    7. Aguardar o build (2 a 3 minutos)
    8. Copiar a URL gerada (ex: visionsearch.vercel.app)

  Apos o deploy:
    No Supabase > Authentication > URL Configuration, adicionar
    a URL da Vercel em "Redirect URLs":
      https://visionsearch.vercel.app  (ou a URL que a Vercel gerar)

  No Google Cloud Console:
    Adicionar a URL da Vercel em Authorized redirect URIs:
      https://visionsearch.vercel.app  (ou a URL gerada)

  Como verificar:
    Acessar a URL da Vercel no navegador.
    O app deve carregar normalmente e o login com Google deve funcionar.
